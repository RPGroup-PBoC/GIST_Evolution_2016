Genetic Drift

In this script we will simulate Burin's experiment where 16 flies are placed in the same vile and allowed to reproduce. 16 new flies will be passed to a new vile and they will be allowed to reproduce again.

For our simulation we will make some simplifying assumptions:

  1. There is a constant population size with no overlapping generations. This means that the original 16 flies can reproduce only once and then die out.
  2. There are no mutations nor recombination that will generate a new allele.
  3. Both alleles have the same fitness.
  4. We do not take into account sex for the selection of parents.

With the limitations that these assumptions might have on our simulation it can still give us a lot of intutition about how genetic drift can be a driving force for evolution. Hopefully after this simulation you will be convince that the classic picture where Darwinian natural selection is the only driving force for the evolution of populations is not entirely right. There are other forces such as genetic drift that can shape the evolutionary history of small populations.

For our simulation the first thing we need to do is to define the population size and the number of generations to run the simulation

% Define population size

pop_size = 16; % as in Burin's experiment

% Define the number of generations to run the simulation

n_gen = 20; % also as in Burin's experiment

Next we need to initialize arrays to keep track of the flies genotypes over time. If we have 16 flies we need two numbers to define their genotype since the flies are diploid, i.e. they have two copies of each locus. So by defining two arrays we can keep track of one generation ( means that it is a row vector with entries). But since we want to keep track of all generations we will instead define two n_genmatrices where on each row we will register the allele of each of the 16 flies on each generation.

% initialize matrices to keep track of each fly's locus

locus_1 = zeros(n_gen, pop_size);

locus_2 = zeros(n_gen, pop_size);

Now that we initialize our arrays to keep track of the locus let's modify the first row, i.e. the first geneartion with the initial conditions of the experiment. We know that Burin started the experiment with 8 heterozygous flies. This means that each fly had a copy of allele 1 and a copy of allele 2. Since we are not distinguishing between locus we can for simplicity say that all the flies had allele 1 at locus 1 and all flies had allele 2 at locus 2.

% add alleles of generation 1.

locus_1(1, :) = ones(1, pop_size);

% for allele 2 we also use the function ones and simply multiply by 2.

locus_2(1, :) = ones(1, pop_size) * 2;

Running the simulation

Now comes the nasty part of the code. We need to do the following steps that will involve a bunch of nested for loops

  1. Loop through the generations.
  2. Loop through each of the new 16 baby flies that will go into the next generation.
  3. For each of these baby flies select two random parents.
  4. From this pair of random parents select 1 allele from each to give it to the new fly.
  5. Register each of the alleles in our locus_1 and locus_2 matrices.

Having this clear let's code it!

% STEP 1: loop through generations

for i=2:n_gen

% STEP 2: loop through each of the newborn flies

for j=1:pop_size

% STEP 3: select random parents

parents = randsample(1:pop_size, 2);

% INTERMEDIATE STEP:

% pick out the parents genotype from our locus_1 and locus_2 matrices

p_allele_1 = [locus_1(i-1, parents(1)) ...

locus_2(i-1, parents(1))];

p_allele_2 = [locus_1(i-1, parents(2)) ...

locus_2(i-1, parents(2))];

% STEP 4: select randomly the allele for the baby fly

l_1 = randsample(p_allele_1, 1);

l_2 = randsample(p_allele_2, 1);

% STEP 5: save these alleles to keep track of them

locus_1(i, j) = l_1;

locus_2(i, j) = l_2;

end

end

Number of generations for allele fixation

When we are dealing with a small population undergoing genetic drift, such as the flies in Burin's experiment, it is possible that one of the alleles gets fixed. This means that all of the flies in a vile will turn homozygous and, under the assumption that there is no mutation to recover the lost allele, it is impossible to go back to have to the original state. But the question we want to ask is: on average how many generations need to pass before one allele gets fixed, and how does this mean number scales with population size?

To tackle this question numerically we can make use of the simulation that we just wrote. But instead of using only one population size (16 flies per vile) we can run it for different experiments (for example 4, 8, 16, 32, 64 and 128 flies per vile). Also since we don't know how many generations we need to run the simulation we need to adapt our code such that instead of running a fixed number of generations it continues running until either allele gets fixed in the population. We then register this number of generations since is the quantity we care about.

Starting with the simple case of only 16 flies

It is a good programmig practice to try to test a code in a simpler version before jumping to the hardcore run. In our case before trying to use multiple population sizes we will first try only with the population size we were using before.

We need to then define the number of viles that we will use in our experiment and initialize an array to save the number of generations it took for each vile to get an allele fixed.

Note that we will re-use some variables names. In general this is never recommended, but in order not to confuse ourselves we will stick to the names we used before.

% define the number of fly viles, i.e.number of experiments.

n_viles = 107;

% just in case re-define population size

pop_size = 16; % as in Burin's exp

% initialize array to save fixation time

n_gens = zeros(1, n_viles);

Now here it comes again! The nasty part of the code. As before let's list the number of steps we need to follow so that we don't get lost in the nested loops. The main difference with this new code and the one we wrote in the previous version is that instead of looping through the number of generations we will use a while loop. What a while loop does is that it continues running until a condition is satisfied. In our case the condition that needs to be satisfied in order for the while loop to stop is that either allele gets fixed in the vile. This could take 5 generations, maybe 20 or maybe 107, but we don't need to worry about it because the loop will keep runnning until the condition is satisfied.

So the steps are:

  1. Loop through the number of viles
  2. Initialize the arrays to keep track of each fly's locus. We do it inside the loop because this time we don't want to keep this vector all the way through the simulation. We will re-write it on each iteration since what we care about is one particular event.
  3. Initialize the boolean variable that will define if the while loop should continue to the next iteration or should it stop. What the variable simply asks is: is allele 1 or allele 2 fixed in the population?
  4. Initialize a so-called counting variable that will keep track of how many interations the while loop has been running.
  5. Start the while loop that will continue until the boolean variable becomes true.
  6. Loop through each of the new baby flies that will go into the next generation.
  7. For each of these baby flies select two random parents.
  8. From this pair of random parents select 1 allele from each to give it to the new fly.
  9. Register each of the alleles in our locus_1 and locus_2 arrays.
  10. After generating the baby flies ask again the question: is allele 1 or allele 2 fixed in the population? to decide if the loop should continue or should stop.
  11. Update the counting variable
  12. If the loop stop save the number of generations that the simulation ran for

And with this significantly more substancial list of things to do let's just atack the problem! Make sure you go back to this list as many times as you need to understand the logic of the code. It is not a simple code so if you are not able to understand it the first time don't worry; go again through the list and the code and think on each step why are we doing what we are doing.

% STEP 1: loop through the number of viles

for i=1:n_viles

% STEP 2: initialize arrays to keep track of each fly's locus.

loci_1 = ones(1, pop_size);

loci_2 = ones(1, pop_size) * 2;

% STEP 3: Initialize the boolean variable

fix_bool = all((loci_1 == 1 & loci_2 == 1) |...

(loci_1 == 2 & loci_2 == 2));

% STEP 4: Initialize a counting variable

n_gen = 1;

% STEP 5: Start the while loop

while fix_bool == false;

% STEP 6: Loop through each of the new baby flies

for j=1:pop_size

% STEP 7: select parents without replacement

parents = randsample(1:pop_size, 2);

% INTERMEDIATE STEP:

% pick out the parents genotype from our locus_1 and locus_2 arrays

p_allele_1 = [loci_1(parents(1)) loci_2(parents(1))];

p_allele_2 = [loci_1(parents(2)) loci_2(parents(2))];

% STEP 8: randomly choose alleles from the parents

l_1 = datasample(p_allele_1, 1);

l_2 = datasample(p_allele_2, 1);

% STEP 9: save alleles to keep track of them

loci_1(j) = l_1;

loci_2(j) = l_2;

end

% STEP 10: ask again the question

fix_bool = all((loci_1 == 1 & loci_2 == 1) |...

(loci_1 == 2 & loci_2 == 2));

% STEP 11: Update the counting variable

n_gen = n_gen + 1;

end

% STEP 12: If the loop stop save the number of generations

n_gens(i) = n_gen;

end

The simulation seems to have run without any problem. A good way to visualize the result is to just plot the histogram of the number of generations it took on each vile to reach fixation.

hist(n_gens)

xlabel('Number of generations to fixation', 'FontSize', 20)

ylabel('Frequency', 'FontSize', 20)

Also we can print relevant quantities such as the mean and standard deviation

disp(['Mean number of generations: ' num2str(mean(n_gens))]);

Mean number of generations: 13.1963

disp(['Standard deviation of number of generations: ' num2str(std(n_gens))]);

Standard deviation of number of generations: 7.5304

This is quite interesting! on average it took 13 generations for an allele to get fix, but there is a bit of a tail towards higher numbers.

Including different population sizes

Now that we went through this complex code for one population size, scaling it to include different number of flies per experiment is very simple! In our list of things to do we only need to add one step that we will call step 0. loop through the different number of population sizes. And with that extra step we can literally copy and paste the code that we just wrote. We just need to generate a matrix to save each iteration result for each experiment on each populaation size.

% define number of fly vials

n_vials = 107;

% define population size

pop_size_array = [4 8 16 32 64 128];

% initialize array to save # of gen to fix

n_gen_mat = zeros(length(pop_size_array), n_vials);

% STEP 0: loop through the number of population sizes

for p=1:length(pop_size_array)

%NOTE that we don't close the semicolon here. We want the pop_size variable

%to be printed in the terminal to know that the code is running.

pop_size = pop_size_array(p)

% STEP 1: loop through the number of viles

for i=1:n_viles

% STEP 2: initialize arrays to keep track of each fly's locus.

loci_1 = ones(1, pop_size);

loci_2 = ones(1, pop_size) * 2;

% STEP 3: Initialize the boolean variable

fix_bool = all((loci_1 == 1 & loci_2 == 1) |...

(loci_1 == 2 & loci_2 == 2));

% STEP 4: Initialize a counting variable

n_gen = 1;

% STEP 5: Start the while loop

while fix_bool == false;

% STEP 6: Loop through each of the new baby flies

for j=1:pop_size

% STEP 7: select parents without replacement

parents = randsample(1:pop_size, 2);

% INTERMEDIATE STEP:

% pick out the parents genotype from our locus_1 and locus_2 arrays

p_allele_1 = [loci_1(parents(1)) loci_2(parents(1))];

p_allele_2 = [loci_1(parents(2)) loci_2(parents(2))];

% STEP 8: randomly choose alleles from the parents

l_1 = datasample(p_allele_1, 1);

l_2 = datasample(p_allele_2, 1);

% STEP 9: save alleles to keep track of them

loci_1(j) = l_1;

loci_2(j) = l_2;

end

% STEP 10: ask again the question

fix_bool = all((loci_1 == 1 & loci_2 == 1) |...

(loci_1 == 2 & loci_2 == 2));

% STEP 11: Update the counting variable

n_gen = n_gen + 1;

end

% STEP 12: If the loop stop save the number of generations

n_gen_mat(p, i) = n_gen;

end

end
